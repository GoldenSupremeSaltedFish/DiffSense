核心特性

AST语义分析

超越传统模式匹配，通过抽象语法树进行深度语义理解，精准识别代码变更中的潜在风险。

增量式扫描

仅分析代码差异部分，显著提升分析效率，适用于高频提交的CI/CD流水线集成。

信号驱动架构

检测与评估分离的设计模式，支持灵活扩展自定义风险规则，适应多样化业务场景。

自适应分析深度

根据代码变更规模动态调整分析策略，在性能与准确性之间实现最优平衡。

快速开始

1

在CI/CD中引入DiffSense

# GitHub Actions 示例
- name: Run DiffSense Analysis
  uses: diffsense/action@v1
  with:
    github-token: ${{ secrets.GITHUB_TOKEN }}
    
# GitLab CI 示例  
- stage: analyze
  image: diffsense/dock:latest
  script:
    - diffsense analyze --ci
2

配置风险规则（可选）

# 在项目根目录创建 .diffsense.yml
rules:
  - name: concurrency-risk
    enabled: true
  - name: security-validation  
    enabled: true

DiffSense通过预构建的Docker镜像提供服务，无需手动下载或打包。支持GitHub Actions、GitLab CI、Jenkins等主流CI/CD平台。
